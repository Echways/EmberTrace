# EmberTrace.Abstractions

Минимальная сборка с атрибутами, которые используются в приложениях и читаются `EmberTrace.Generator`.

Цель — держать атрибуты отдельно от runtime и генератора, чтобы:
- не тащить лишние зависимости,
- удобно подключать к любому проекту,
- иметь стабильный контракт разметки.

## Атрибуты

### `TraceIdAttribute`
Маркирует символ (обычно метод), для которого генератор должен:
- вычислить стабильный `int id` (хэш или явный override),
- добавить запись в `TraceIds`,
- добавить метаданные (name/category) для отчётов.

### `TraceNameAttribute`
Задаёт человекочитаемое имя, которое попадёт в отчёты и экспорт.
Используется только в cold path.

### `TraceCategoryAttribute`
Необязательная категория/подсистема для группировки и фильтрации в отчётах.
