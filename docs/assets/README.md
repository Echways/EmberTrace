# Assets

Статические файлы (скриншоты/диаграммы), используемые в документации.

## Правила

- **Имена файлов**: `kebab-case`, отражают смысл и место использования (например, `getting-started-first-trace.png`).
- **Формат**: PNG для скриншотов, SVG для схем/диаграмм (если есть исходники).
- **Размер**: ориентируйся на 1200–1600px по ширине, чтобы читалось и в GitHub, и на будущём docs-site.
- **Гигиена**: убирай приватные данные (пути, имена машин, токены), оставляй только важные панели/фрагменты.
