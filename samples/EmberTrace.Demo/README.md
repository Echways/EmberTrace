# EmberTrace.Demo

Минимальный пример, который показывает весь путь:
1) разметка `[TraceId]`,
2) использование `TraceIds` в `Profiler.Scope(...)`,
3) запуск/остановка сессии,
4) обработка,
5) вывод отчёта.

## Что тут есть
- `Program.cs` — запуск сессии и демонстрационный workload.
- `Workloads/` — небольшие сценарии, которые удобно менять:
    - синтетический CPU
    - IO/ожидания
    - “physics-like” циклы

## Как запускать
- Запусти проект как обычное консольное приложение.
- На выходе должен появиться отчёт (Top Self/Total и дерево вызовов).

## На что смотреть
- Генерация `TraceIds.g.cs` и `TraceMetadata.g.cs` должна происходить автоматически.
- `Profiler.Scope(TraceIds.X)` не должен аллоцировать память в горячем цикле.
